# 🍽️ 卡路里追蹤教練管理系統

> 健身教練管理學員的飲食、訓練、體重記錄系統 — Streamlit + Gemini 2.5 Flash + Google Sheets + Firebase Storage

**上線版本**：[https://worldgymzoe-caloriesoe.streamlit.app/](https://worldgymzoe-caloriesoe.streamlit.app/)
**Repository**：[hungwei62-blip/calorie-tracker](https://github.com/hungwei62-blip/calorie-tracker)

---

## 📌 專案定位更新

本系統從「多人共享卡路里追蹤」升級為「健身教練管理系統」。

### 系統角色

| 角色 | 說明 |
|---|---|
| **系統管理員** | 建立教練帳號 |
| **教練** | 管理所有學員（目前 1 位教練） |
| **學員** | 由教練建立，記錄自己的飲食、訓練、體重（上限 50 人） |

---

## 🎯 新功能需求確認

### 學員端功能

| # | 功能 | 說明 |
|---|---|---|
| 1 | 每日飲食記錄（簡易模式） | 蛋白質 (g)、熱量 (kcal)、飲水量 (ml) |
| 2 | 每日飲食記錄（完整模式） | 蛋白質、碳水、脂肪、熱量、飲水量 |
| 3 | Mode 切換 | 學生自己選擇簡易或完整模式 |
| 4 | AI 食物計算 | Gemini 自動分析文字/圖片並填入紀錄 |
| 5 | 體重紀錄 | 每日/每週輸入 + 體重趨勢圖 |
| 6 | 訓練記錄 | 學生填寫日期 + 項目（背/胸/腿/核心/有氧） |
| 7 | 每日完成率 | 顯示飲食/水量/訓練 完成狀態 |

### 教練端功能

| # | 功能 | 說明 |
|---|---|---|
| 1 | 所有學員總覽 | 首頁一覽表（姓名/蛋白質/熱量/水量/訓練/體重/完成率） |
| 2 | 新學員入職流程 | TDEE 問卷 → 自動計算目標 |
| 3 | 修改學員目標 | 教練可手動調整學員目標 |
| 4 | 自動完成率計算 | 蛋白質/熱量/水量/訓練 完成率 |
| 5 | 學員歷史頁面 | 從總覽點選學員進入：圖表（體重/飲食組成/攝取趨勢/訓練）、教練備註、PDF + CSV 匯出 |
| 6 | 教練備註 | 獨立 Notes 工作表，每次修改新增一列，含時間戳記 |

### 資料夾結構

``
calorie-tracker/
  app.py                       # Streamlit 主程式
  services/
    auth.py                    # 登入驗證（教練/學員）
    gemini.py                  # Gemini AI 分析
    sheets.py                   # Google Sheets 操作
    firebase.py                 # Firebase 圖片上傳
    metrics.py                  # 熱量/營養計算
  config/
    secrets.example.toml        # 設定檔範例
  docs/
    SETUP_GCP_FIREBASE.md       # GCP/Firebase 設定教學
    DEPLOY_CHECKLIST.md         # 部署檢查清單
  tools/
    json_to_toml.py             # JSON 轉 TOML 工具
  requirements.txt              # Python 套件
  README.md                     # 本文件
``

### Google Sheets 工作表規劃（預計）

| 工作表 | 用途 |
|---|---|
| Users | 教練與學員帳號資訊 |
| Goals | 各學員的營養目標（蛋白質/碳水/脂肪/熱量/水量） |
| Records | 學員每日飲食記錄 |
| Weight | 學員體重記錄 |
| Training | 學員訓練記錄（背/胸/腿/核心/有氧） |
| Notes | 教練對學員的觀察備註（時間戳記 / 學員 ID / 教練 ID / 內容） |
